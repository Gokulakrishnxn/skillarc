## Overview

A read-only file tool used by coding agents and document-grounded assistants. Confines reads to a workspace root, enforces a max byte budget, and refuses binary files politely.

<Callout variant="info" title="Pair with a writer">
For agents that need to edit files, ship this alongside a separate `file_write` tool with explicit allowlists — keeping read and write split makes auditing easier.
</Callout>

## Usage

```json:tool.json
{
  "name": "file_read",
  "description": "Read a UTF-8 text file from the workspace. Returns its content and a checksum so the caller can detect drift between reads.",
  "parameters": {
    "type": "object",
    "required": ["path"],
    "properties": {
      "path": {
        "type": "string",
        "description": "Workspace-relative path. Absolute paths are rejected."
      },
      "start_line": { "type": "integer", "minimum": 1 },
      "end_line": { "type": "integer", "minimum": 1 },
      "max_bytes": {
        "type": "integer",
        "default": 65536,
        "description": "Truncate after this many bytes; the response will set `truncated: true`."
      }
    }
  }
}
```

## Parameters

| Field | Type | Notes |
| --- | --- | --- |
| `path` | string | Resolved relative to `WORKSPACE_ROOT`. |
| `start_line` / `end_line` | int | Optional 1-indexed inclusive range. |
| `max_bytes` | int | Hard cap; binary content is rejected. |

## Examples

```typescript:executor.ts
import { promises as fs } from "node:fs";
import { resolve, relative, join } from "node:path";
import { createHash } from "node:crypto";

const ROOT = resolve(process.env.WORKSPACE_ROOT!);

export async function readFile(args: {
  path: string;
  start_line?: number;
  end_line?: number;
  max_bytes?: number;
}) {
  const target = resolve(join(ROOT, args.path));
  const inside = relative(ROOT, target);
  if (inside.startsWith("..")) throw new Error("Path escapes workspace.");

  const raw = await fs.readFile(target);
  if (raw.includes(0)) throw new Error("Refused: binary file.");

  const text = raw.toString("utf8");
  const lines = text.split("\n");
  const start = (args.start_line ?? 1) - 1;
  const end = args.end_line ?? lines.length;
  const slice = lines.slice(start, end).join("\n");
  const truncated = slice.length > (args.max_bytes ?? 65536);

  return {
    path: args.path,
    content: truncated ? slice.slice(0, args.max_bytes) : slice,
    truncated,
    sha256: createHash("sha256").update(text).digest("hex"),
  };
}
```

## Compatible LLMs

Universally compatible — every function-calling model can invoke this tool reliably. Smaller models occasionally pass absolute paths; the executor's path-escape guard turns those into a graceful error the model can correct.
